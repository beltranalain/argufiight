### Step 3: Set Environment Variables (5 minutes)

In your hosting platform, set these:

```env
DATABASE_URL=file:./prisma/prod.db
# OR if using PostgreSQL:
# DATABASE_URL=postgresql://user:pass@host:5432/dbname

AUTH_SECRET=your-strong-secret-here-minimum-32-characters-long
NEXT_PUBLIC_APP_URL=https://yourdomain.com
NODE_ENV=production
```

**Generate AUTH_SECRET:**
```bash
# Run this to generate a secure secret:
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Step 4: Database Setup (10 minutes)

**Option A: SQLite (Simplest - for starting)**
- [ ] Database file will be created automatically
- [ ] Run migrations: `npx prisma migrate deploy` (if using migrations)
- [ ] Or: `npx prisma db push` (if using db push)

**Option B: PostgreSQL (Recommended for production)**
- [ ] Set up PostgreSQL database (Vercel Postgres, Supabase, Railway, etc.)
- [ ] Get connection string
- [ ] Update `DATABASE_URL` in environment variables
- [ ] Run migrations: `npx prisma migrate deploy`
